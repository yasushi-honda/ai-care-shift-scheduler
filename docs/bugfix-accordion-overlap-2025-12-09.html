BUG-020: アコーディオン展開時のUI重複修正記録

スタッフ情報展開時に下部コンテンツと重複表示される問題

重大度: 中 カテゴリ: UI/UX 修正済み
問題概要

症状

スタッフ情報のアコーディオンを展開すると、下部のコンテンツ（シフト種別設定、勤務できない日、スタッフ削除ボタン等）と重なって表示される。

視覚的問題:
- スタッフ「伊藤真理」を展開
- 保有資格チェックボックスの下に「シフト種別設定」セクションが重複表示
- 「早番」「日勤」「夜勤」のラベルがスタッフ情報内に侵入
- 「スタッフを削除」ボタンが正しく配置されない

影響範囲

  • Accordionコンポーネントを使用する全てのセクション
  • スタッフ情報設定（特に複数スタッフ展開時）
  • シフト種別設定
  • 休暇残高管理
  • 事業所のシフト要件設定
根本原因

問題のコード

// components/Accordion.tsx:33
<div className={`transition-all duration-300 ease-in-out
  ${isOpen ? 'max-h-screen' : 'max-h-0 overflow-hidden'}`}>

技術的詳細

問題点 詳細
max-h-screenの制限 100vh（ビューポート高さ）は固定値であり、コンテンツ量とは無関係
overflowの適用タイミング 閉じた状態ではoverflow-hiddenだが、開いた状態では未指定（visible）
ネストされた構造 StaffSettings内の個別スタッフ展開も同様の動作をする
修正内容

CSS Gridベースのアニメーションに変更

修正前（max-height方式）

<div className={`
  transition-all duration-300
  ${isOpen
    ? 'max-h-screen'
    : 'max-h-0 overflow-hidden'}
`}>
  <div className="p-4 ...">
    {children}
  </div>
</div>

修正後（CSS Grid方式）

<div className={`
  grid transition-[grid-template-rows]
  duration-300
  ${isOpen
    ? 'grid-rows-[1fr]'
    : 'grid-rows-[0fr]'}
`}>
  <div className="overflow-hidden">
    <div className="p-4 ...">
      {children}
    </div>
  </div>
</div>

方式比較

項目 max-height方式 CSS Grid方式
コンテンツ量対応 固定値（100vh）で制限 自動で1frに拡張
アニメーション 0→固定値（不自然な場合あり） 0fr→1fr（自然な展開）
overflow制御 開いた状態でvisible（重複原因） 常にhiddenでclip
ネスト対応 親子の高さ計算が競合 各グリッドが独立計算
修正のポイント:
  • grid-rows-[0fr]: 高さ0（閉じた状態）
  • grid-rows-[1fr]: コンテンツに応じた高さ（開いた状態）
  • 内部にoverflow-hiddenを配置して確実にクリッピング
  • transition-[grid-template-rows]でスムーズなアニメーション

教訓

1. max-heightアニメーションの限界

  • コンテンツ量が可変の場合、固定値のmax-heightは不適切
  • max-h-screenでも内部コンテンツが多いと重複が発生

2. CSS Grid grid-rows-[0fr]/[1fr]パターン

  • コンテンツ量に関係なく自然なアニメーションが可能
  • 内部のoverflow-hiddenと組み合わせることでクリッピングも確実

3. ネストされたUI構造の考慮

  • アコーディオン内にさらに展開可能なUIがある場合、高さ計算が複雑になる
  • 各レベルで独立して機能するCSS手法を選択すべき
