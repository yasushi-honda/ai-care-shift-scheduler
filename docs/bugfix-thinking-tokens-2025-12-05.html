BUG-003: Gemini 2.5 Flash 思考トークン問題修正

概要
発見日2025-12-05
修正完了日2025-12-05
重要度Critical
影響本番環境でAIシフト生成が完全に動作不能
前提バグBUG-002（propertyOrdering追加）修正後に発覚
エラー内容
finishReason: 'MAX_TOKENS'
thoughtsTokenCount: 8191
Response text length: 0
❌ JSON Parse Error: SyntaxError: Unexpected end of JSON input

ポイント: Gemini 2.5 Flashの「思考モード」が8191トークン消費し、maxOutputTokens: 8192を使い切り、出力用トークンが残らなかった

問題発生フロー
sequenceDiagram participant User as ユーザー participant CF as Cloud Functions participant Gemini as Gemini 2.5 Flash User->>CF: AIシフト生成リクエスト CF->>Gemini: 骨子生成リクエスト
(maxOutputTokens: 8192) Note over Gemini: 思考モード開始
thoughtsTokenCount: 8191 Note over Gemini: 出力用トークン残り: 1
→ 出力生成不可 Gemini-->>CF: finishReason: MAX_TOKENS
text: ""（空） CF-->>User: HTTP 500 Error
根本原因
flowchart TD A[maxOutputTokens: 8192] --> B[Gemini思考モード開始] B --> C[思考に8191トークン消費] C --> D[出力用トークン: 1のみ] D --> E[出力生成を断念] E --> F[finishReason: MAX_TOKENS] F --> G[空のレスポンス] style A fill:#f39c12,color:#fff style C fill:#e74c3c,color:#fff style G fill:#e74c3c,color:#fff

既知の問題:

"If MAX_TOKENS finish reason is triggered, the response text is empty, making debugging very difficult."
- googleapis/python-genai Issue #782
修正内容

Before（修正前）

maxOutputTokens: 8192,

After（修正後）

maxOutputTokens: 65536,  // Gemini 2.5 Flash thinking mode uses tokens from this budget

結果 MAX_TOKENS + 空 正常完了

教訓: BUG-002で追加したデバッグログがBUG-003の即時発見に貢献

学び・教訓
  1. デバッグログの価値 - BUG-002で追加したログがBUG-003の即時発見に貢献
  2. Gemini 2.5の新機能に注意 - 思考モードは強力だが、トークン消費に影響
  3. 空レスポンスには複数の原因 - responseLength: 0だけでは原因特定不可、finishReason確認が必須
  4. 余裕を持った設定 - トークン制限は余裕を持って設定（8192 → 65536）
