BUG-008: thinkingBudget制限修正記録

Gemini 2.5 Flash思考トークン過消費によるMAX_TOKENSエラー

重大度: 高 修正済み
問題概要

症状

コンソールエラー:

HTTPレスポンス:
500 Internal Server Error

発生条件

  • デモ環境でシフト生成を実行
  • スタッフ数: 12名（常勤8名 + パート4名）
  • Phase 46で追加したパート職員4名が原因でトリガー

maxOutputTokens設定
根本原因: Gemini 2.5 Flashの思考モードは maxOutputTokens の予算から思考トークンを消費する。 思考が全予算を使い切ると、出力用トークンが0になりJSONレスポンスが空になる。

修正内容

解決策: thinkingBudget制限の追加

Gemini 2.5 Flashには thinkingConfig.thinkingBudget パラメータがあり、思考トークンの上限を設定できる（範囲: 0-24576）。

修正コード (phased-generation.ts): 
generationConfig: {
  maxOutputTokens: 65536,
  // 思考トークンを制限（12名スタッフで65535トークン使い切りエラー対策）
  thinkingConfig: {
    thinkingBudget: 16384,  // 思考に16K、残りを出力に使用
  },
}

学びと再発防止

重要な学び

  1. 思考モードはトークン予算を共有する: maxOutputTokens を増やしても、思考がすべて消費すれば出力は空になる
  2. スケールに注意: 8名では問題なくても、12名では破綻することがある
  3. 明示的な制限が重要: デフォルト（制限なし）に頼らず、明示的にbudgetを設定する

推奨設定ルール

// 常にthinkingBudgetを明示的に設定すること
thinkingConfig: {
  thinkingBudget: 16384,  // 最大24576、通常16384推奨
}
