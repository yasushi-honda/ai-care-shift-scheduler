Phase 42.2: デモログイン機能

Cloud Function認証によるセキュアなワンクリックデモアクセス

完了

概要

クライアント（見込み顧客）にアプリケーションを簡単に試用してもらうため、ワンクリックでデモ環境にアクセスできる機能を実装しました。

目的

  • クライアントがワンクリックでデモ環境にアクセス
  • 既存のデモデータ（サンプル介護施設）で機能を体験
  • 営業・デモ活動の効率化

セキュリティ

  • 認証情報はサーバーサイドでのみ保持
  • Cloud Functionでカスタムトークン発行
  • viewer権限のみ（データ変更不可）
  • 特定ドメインのみCORS許可

デモログインの使い方

ワンクリックでデモ環境にアクセス
ログインページの「デモアカウントでログイン」ボタンをクリックするだけ！
アカウント作成やパスワード入力は不要です。

手順

  1. https://ai-care-shift-scheduler.web.app にアクセス
  2. ログインページで「デモアカウントでログイン」ボタンをクリック
  3. サンプル介護施設のシフト管理画面が表示されます

セキュリティ設計

CORS制限

以下のドメインのみCloud Function呼び出し可能：

  • ai-care-shift-scheduler.web.app
  • ai-care-shift-scheduler.firebaseapp.com
  • localhost:5173（開発用）
  • localhost:3000（E2Eテスト用）

権限制限

  • viewer権限のみ - シフトの閲覧は可能、変更は不可
  • デモ施設のみ - demo-facility-001へのアクセス限定
  • 本番データ隔離 - 他の施設データへのアクセス不可

Cloud Function コード抜粋

export const demoSignIn = onRequest({
  cors: [
    'https://ai-care-shift-scheduler.web.app',
    'https://ai-care-shift-scheduler.firebaseapp.com',
    'http://localhost:5173',
    'http://localhost:3000',
  ],
  timeoutSeconds: 30,
  memory: '256MiB',
}, async (req, res) => {
  // デモユーザー作成・カスタムトークン発行
  const customToken = await auth.createCustomToken(DEMO_USER_UID);
  res.status(200).json({ customToken });
});

IAM権限要件（必須）

重要: Cloud Functionでカスタムトークンを発行するには、サービスアカウントに Service Account Token Creatorロールが必要です。
サービスアカウント 必要なロール
ai-care-shift-scheduler@appspot.gserviceaccount.com roles/iam.serviceAccountTokenCreator

設定方法: GCP Console → IAM → サービスアカウントに「Service Account Token Creator」ロールを追加
