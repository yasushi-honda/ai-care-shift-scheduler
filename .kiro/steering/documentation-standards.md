# Documentation Standards - ドキュメント作成基準

**最終更新**: 2025-12-29

---

## 原則

**テキスト + Mermaid図の併用**

- テキストドキュメント: 詳細・理由・コンテキスト
- Mermaid図: 全体像・構造・関係性
- 両者を相互参照して補完

---

## 記録が必要なタイミング

1. **Phase完了時**（Phase 5, 10, 15など節目）
2. **重大なバグ修正後**（設計判断の変更を伴う場合）
3. **本番環境デプロイ後**（初回、大規模変更時）
4. **アーキテクチャ変更後**
5. **月次レビュー時**

---

## ファイル命名規則

| 種別 | パターン |
|------|----------|
| Phase検証 | `phase[N]-verification-YYYY-MM-DD.md` |
| バグ修正 | `bugfix-[description]-YYYY-MM-DD.md` |
| 開発状況 | `development-status-YYYY-MM-DD.md` |
| アーキテクチャ図 | `architecture-diagram-YYYY-MM-DD.md` |

**保存場所**: `.kiro/` または `.kiro/specs/[feature]/`

---

## テキストドキュメント テンプレート

```markdown
# [タイトル]

**更新日**: YYYY-MM-DD
**仕様ID**: [spec-name]
**Phase**: [phase-number]

## 概要
[何が行われたか、なぜ行われたか]

## 詳細内容
[技術的決定、実装方法]

## 検証結果
[動作確認結果、エビデンス]

## 影響分析
[変更による影響範囲]

## 今後の対応
[次のステップ、未完了項目]

## 関連ドキュメント
[リンク]
```

---

## Mermaid図の種類

| 図の種類 | 用途 |
|----------|------|
| ガントチャート | Phase実装進捗 |
| システムアーキテクチャ図 | コンポーネント構成 |
| シーケンス図 | 処理フロー |
| ER図 | データモデル |
| フローチャート | 判断分岐 |

---

## 実装時の注意

- **記録は実装の一部**: コードと同様に重要
- **リアルタイム記録**: 完了時に即座に記録
- **相互参照**: テキスト ↔ 図をリンクで結びつける
